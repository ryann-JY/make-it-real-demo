# Make It Real 用户需求与产品价值定位

## 一、SMB 用户分型、核心需求与 MIR 价值

SMB 不应按副业、全职或设备数量分类。更稳定的标准是：

> 订单为什么发生，客户究竟在为什么付钱。

| SMB 类型 | 定位 | 代表账号 | 核心任务与需求 | MIR 可以提供的核心价值 |
|---|---|---|---|---|
| **场景商品经营型** | **P0 核心消费用户** | [CustomUVSticker](https://www.etsy.com/shop/CustomUVSticker) | 围绕节日、人群和消费场景选择商品，以低库存、小批量方式测试市场 | 提供有明确人群、场景和真实成品依据的商品机会，帮助卖家初步评估下一款值得测试的商品。例如，母亲节前参考花卉祝福木质挂牌的成本、耗时和真实成品，再决定是否小批量上新 |
| **参数化个性商品型** | **P0 核心消费用户** | [Intricut Creations](https://www.etsy.com/shop/IntricutCreations) | 高效处理姓名、照片、日期、Logo 等个性化订单，降低沟通、设计和出错成本 | 将高频定制需求转化为清晰、可预览、可重复的制作方案。例如，卖家输入客户提供的宠物照片和姓名，即可生成冰箱贴预览与可打印设计 |
| **需求驱动定制服务型** | **P1 选择性支持** | [JHCreativeCo](https://www.etsy.com/shop/JHCreativeCo) | 判断客户需求能否制作，并获取报价与交付所需的技术依据 | 提供真实材料、参数和制作案例，辅助技术可行性判断。例如，接到亚克力婚礼桌牌询价时，可参考尺寸限制、定位制具、耗时和真实效果；报价与交期仍由商家决定 |
| **创作者/IP 驱动型** | **战略供给用户** | [Ten Hundred](https://tenhundredart.com/) | 将原创作品延展到实体商品，同时保持作者归属和风格完整性 | 提供UV载体测试、原创展示和署名能力，吸引创作者贡献高审美作品与商品化案例。例如，比较同一插画在画布、木板和亚克力上的真实表现 |

四类 SMB 共同需要：

```text
发现并初步评估商品机会 → 形成可售方案 → 判断商业可行性 → 实现稳定生产 → 建立信任并被发现
```

## 二、Maker 用户分型、核心需求与 MIR 价值

Maker 不应按照浏览内容或是否使用模板分类。更稳定的标准是：

> 用户为什么愿意投入时间、材料和精力完成一次制作。

| Maker 类型 | 定位 | 代表账号 | 核心任务与需求 | MIR 可以提供的核心价值 |
|---|---|---|---|---|
| **生活创造型** | **P0 核心消费用户** | [Evan & Katelyn](https://www.evanandkatelyn.com/about) | 为自己、家庭、朋友或生活空间制作有用途、有情感意义的物品 | 推荐与人物、兴趣和生活场景相关，并且真实可完成的创意。例如，为朋友生日找到一个两小时内可完成的照片纪念牌，并提前了解材料、难度和真实效果 |
| **探索学习型** | **P0 核心消费用户** | [Uncle Jessy](https://www.youtube.com/@UncleJessy) | 通过测试设备、材料和参数理解 UV 打印边界，获得能力与掌控感 | 整理材料测试、参数对比、失败原因和实际结果，减少无效试错。例如，打印玻璃前先比较附着处理、白墨层数和后处理方式，再选择实验起点 |
| **审美表达型** | **P1 选择性支持** | [Nerdforge](https://nerdforge.store/about) | 将原创想法和个人视觉语言转化为实体作品，保持创作控制和风格完整 | 不干预创意，提供原创作品实体化所需的UV工艺支持和展示空间。例如，帮助艺术家判断纹理高度、光泽和材料对原创插画色彩的影响 |
| **社区贡献型** | **战略供给用户** | [3D Printing Nerd](https://www.joeltelling.com/) | 分享项目、测试和教程，帮助他人并建立社区影响力 | 让经验能够被记录、引用和复用，并通过署名、反馈和激励换取持续知识供给。例如，一次木板附着测试可以被其他用户验证并持续积累专业认可 |

四类 Maker 共同需要：

```text
找到有意义的创作目标 → 保持创作自主 → 建立完成作品的信心 → 探索并掌握材料工艺 → 获得成就、认可与连接
```

## 三、用户为什么需要 MIR

UV 打印机提供了大量制作可能性，Studio 解决设计与打印执行，但两者之间仍然缺少一层内容和决策系统：

```text
什么值得做？
→ 怎样形成自己的方案？
→ 需要什么材料和工艺？
→ 别人是否真实打印成功过？
→ 这些经验来自谁，是否可信？
```

MIR 需要填补的正是这段空白。

MIR 的长期目标是：

> **构建个性化创意打印生态，让用户能够发现创意、形成个性化方案、完成打印，并持续复用和分享真实经验。**

当前阶段，MIR 聚焦 UV 打印场景，定位为：

> **基于真实作品的个性化 UV 创意打印方案社区。**

用户来到 MIR，是为了：

> **找到一个被真实验证、确实能够打印出来的 UV 项目，再通过平台的个性化工具，把它改成自己独一无二的作品或商品。**

SMB 与 Maker 的具体目标不同，但共同任务是一致的：

> **找到一个已经被做成，并且自己能够复用的 UV 打印方案。**

进入具体场景后，这一共同任务再分别表现为：

- **SMB（Make to Sell）**：找到可以改款、测试和复产的商品方案。
- **Maker（Make for Fun）**：找到可以复刻、学习和个性化的创作方案。

因此，MIR 不必在顶层定位中同时强调“值得卖”和“有意义”。两类价值应在具体内容、筛选方式和使用场景中分别承接。

### MIR 的差异化价值

MIR 最有机会建立三层差异：

1. **不是只看灵感，而是真的打印成功过。**
2. **不是只展示成品，而是包含复现所需的设计、材料和工艺信息。**
3. **不是只能原样照抄，而是能够通过参数化和个性化工具，变成用户自己的作品或商品。**

MIR 不打算成为 UV 打印领域的 Pinterest 或 Etsy，而是要补足当前 UV 打印创作中从“看到创意”到“形成自己的方案并成功做出来”之间的断点。

| 产品 | 用户寻找的核心对象 | 核心承诺 |
|---|---|---|
| **Pinterest** | 创意图片 | 帮我获得灵感 |
| **Etsy** | 可购买的成品 | 我可以直接买到 |
| **MIR** | 可复用的 UV 打印方案 | 真实做成过，我也能复刻或改造 |

MIR 所提供的 UV 打印方案不是单张效果图或单个设计文件，而是一组支持用户判断、复现和个性化的完整信息：

> **UV 打印方案 = 成品目标 + 可编辑设计方法 + 材料工艺 + 真实打印结果**

这意味着 MIR 既不止于创意发现，也不直接售卖最终成品，而是让真实作品背后的设计与打印经验成为可以被下一位用户复用的创作起点。

对不同用户，MIR 的最终价值分别是：

> **对 SMB：帮助其从“设备可以打印什么”，走向“我可以改款、测试和复产什么”。**

> **对 Maker：帮助其从“我看到一个想做的作品”，走向“我能用自己的方式把它成功做出来”。**

MIR 不需要同等服务全部八类用户：

- **P0消费侧重点**：场景商品经营型、参数化个性商品型、生活创造型、探索学习型。
- **P1选择性支持**：需求驱动定制服务型、审美表达型。
- **战略供给侧重点**：创作者/IP驱动型、社区贡献型。

## 四、MIR 通过五类内容资产满足用户需求

| 内容资产 | 核心定义 | 回答的问题 | 承接的用户需求 |
|---|---|---|---|
| **Product Idea** | 聚合商品或创作目标、适用 Skill、Material/IDF 和真实 Project | 什么值得做？ | SMB 发现并初步评估商品机会；Maker 找到有意义的创作目标 |
| **Skill** | 有明确输入、处理逻辑和输出结果的设计生成工作流，不是模板素材 | 怎样形成自己的方案？ | SMB 高效生产个性化商品；Maker 低门槛创作和再创作 |
| **Material/IDF** | 以 Material 为主体，整合尺寸、物理属性和从属于该材料的 Interactive Design Flow | 这个材料怎样打印？ | SMB 稳定生产；Maker 材料探索和工艺学习 |
| **Project** | 关联作品、Skill、Material、设备和打印记录的具体成果 | 这个方案是否真的做出来过？ | 提供成品效果和真实打印证据，支持可行性判断 |
| **Creator** | 承载内容来源、作品权属、创作能力、个人风格和社区关系 | 谁做的，是否值得信任？ | SMB 建立商业信誉；Maker 建立创作者身份与社区影响力 |

## 五、最终的产品逻辑

```text
Product Idea → Skill → Material/IDF → Project → Creator
```

> **Product Idea 负责组织消费决策，Project 负责提供真实证据，Skill 与 Material/IDF 分别提供创作方法和工艺方法，Creator 负责来源、权属与信任。**

五类资产分别回答：
```text
Product Idea：帮助用户决定做什么
Skill：帮助用户形成自己的方案
Material/IDF：帮助用户理解怎样打印成功
Project：证明这个方案确实被真实做出来
Creator：沉淀作者的能力、风格、知识和信誉
```

产品边界保持明确：

- **MIR 负责**内容发现、方案判断、工艺知识、真实打印信任和作者关系。
- **Studio 负责**实际设计、排版、参数确认和打印执行。
